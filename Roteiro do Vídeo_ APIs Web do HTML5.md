# Roteiro do Vídeo: APIs Web do HTML5
**Seminário de ADS — Uniftec Caxias do Sul**
**Integrantes:** Patrick Brando, Rafael Lima e Pedro

## 1. Informações Gerais e Dinâmica
- **Duração Sugerida:** 10 a 15 minutos (aproximadamente 1:30 a 2:00 min por API).
- **Formato:** Vídeo gravado com captura de tela (apresentação de slides e código).
- **Ferramentas Sugeridas:** OBS Studio para gravação, Canva ou Google Slides para os slides, VS Code para os exemplos de código.
- **Dinâmica do Trio:** Cada integrante apresenta 2 APIs, alternando para manter o vídeo dinâmico.

## 2. Estrutura do Roteiro

### Divisão das APIs (2 por integrante)

| Integrante | APIs |
| :--- | :--- |
| **Integrante 1** | Geolocation + History API |
| **Integrante 2** | Web Storage + Notification |
| **Integrante 3** | Drag and Drop + Web Workers |

> Combinar quem fica com qual par. Cada um apresenta o slide + demonstração prática no código.

### Roteiro cena a cena

| Momento | Descrição da Cena | Conteúdo / Fala Principal |
| :--- | :--- | :--- |
| **Abertura** (0:00 - 1:00) | Slide de título com nomes e logo da Uniftec. | Apresentação do trio, disciplina e o objetivo do vídeo: explorar 6 APIs nativas do HTML5. |
| **API 1: Geolocation** (1:00 - 3:00) | Slide conceitual + Demonstração prática no navegador. | **Conceito:** Como a web acessa a localização. **Demo:** Abrir o exemplo, clicar em "Obter Localização", mostrar o mapa aparecendo com as coordenadas. |
| **API 2: Web Storage** (3:00 - 5:00) | Slide comparativo (Local vs Session) + Demonstração prática. | **Conceito:** Armazenamento no cliente sem cookies. **Demo:** Salvar um nome no localStorage, dar refresh na página e mostrar que o nome continua lá. Depois fazer o mesmo no sessionStorage, fechar a aba e mostrar que sumiu. |
| **API 3: Drag and Drop** (5:00 - 7:00) | Demonstração prática de arrastar elementos. | **Conceito:** Interatividade nativa do HTML5. **Demo:** Abrir o quadro Kanban, arrastar tarefas entre colunas "A Fazer" → "Fazendo" → "Feito". Depois arrastar um arquivo do PC para a zona de upload. |
| **API 4: History API** (7:00 - 9:00) | Slide sobre SPAs + Demonstração da URL mudando. | **Conceito:** Mudar de página sem recarregar. **Demo:** Clicar nos botões Home/Sobre/Contato, mostrar a URL mudando, usar o botão Voltar do navegador e mostrar que funciona. Mostrar o log de navegação. |
| **API 5: Notification** (9:00 - 11:00) | Slide de permissão + Notificação aparecendo no SO. | **Conceito:** Alertas fora do navegador. **Demo:** Clicar em "Solicitar Permissão", aceitar, depois clicar nos botões de exemplo (e-mail, lembrete) e mostrar a notificação aparecendo no sistema. |
| **API 6: Web Workers** (11:00 - 13:00) | Slide sobre threads + Comparação ao vivo. | **Conceito:** Processamento pesado sem travar. **Demo:** Primeiro clicar em "Calcular na Main Thread" e mostrar a animação travando. Depois clicar em "Calcular com Worker" e mostrar que a animação continua rodando. |
| **Conclusão** (13:00 - 14:00) | Slide final com referências e agradecimentos. | Resumo da importância das APIs para a experiência do usuário moderna e encerramento. |
